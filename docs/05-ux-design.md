# 05. UX設計(主要6フロー)

各フローは「ユーザーの状態 → 体験ステップ → 設計上の仕掛け → 成功指標」で定義する。

## 5.1 初回訪問(SNSからの流入が主)

**想定**: Instagramのストーリー/プロフィールリンクからスマホで着地。滞在意思は薄く、3秒で離脱判断。

| ステップ | 体験 | 設計上の仕掛け |
|---|---|---|
| 0–1秒 | ページが即座に表示される | 静的HTML+LCP画像preload。スケルトンやスピナーを見せない |
| 1–3秒 | 「広告サイトではない」と感じる | 白と余白、ポップアップなし、バナーなし。Instagramと同じ写真トーン=「あの人のサイトだ」と認知 |
| 3–30秒 | 着地コンテンツを読む/見る | 着地は多くが記事 or 商品ページ。冒頭1文+写真で内容が分かる構成 |
| 30秒– | もう1本へ | 記事末尾「登場した商品」「次に読む」。行き止まりなし |
| 離脱前 | ブランドを記憶する | フッター前のAbout導線・一貫したロゴとタグライン |

**成功指標**: 直帰率(1ページで離脱)70%未満 / LCP 1.8s未満。
**アンチパターン禁止**: Cookie同意バナー(Cloudflare Web Analyticsはcookieless為不要)、ニュースレターポップアップ、通知許可要求。

## 5.2 商品検索(目的型)

**想定**: 「前に見たあのコーヒーミルなんだっけ」/ Googleから「◯◯ レビュー」で流入。

- 経路A(サイト内): ヘッダー検索アイコン → `/search/` → 入力即時結果(Pagefind)→ 商品ページ。**2タップ+入力で到達**。
- 経路B(検索エンジン): レビュー記事 or 商品ページに直接着地 → 記事なら本文中の ProductEmbed から商品ページへ。
- 経路C(眺め型): カテゴリ → 商品グリッド。フィルタは設けず、カテゴリ×タグの粒度で十分な規模(〜数百商品)を保つ。規模が超えたらV3でファセット検討。

**成功指標**: 検索利用者の商品ページ到達率50%以上。ゼロヒット時も必ず次の一手(人気タグ・カテゴリ)を提示。

## 5.3 レビュー閲覧(信頼形成)

**想定**: 購入検討中。「本当に良いのか、ステマではないか」を見極めに来ている。

| 要素 | 仕掛け |
|---|---|
| 実体験の証明 | 実写真(生活シーン)・使用期間バッジ・購入日。「3ヶ月使った」が最強のコピー |
| 正直さ | 「気になった点」を必ず表示(データ必須項目)。これが他アフィサイトとの決定的差別化 |
| 透明性 | PR表記を隠さない。`/disclosure/` でポリシー明文化 |
| 読みやすさ | 本文はリード→体験→良い点→気になる点→シーンの固定構成。流し読みでも要点が拾える(Good/ConcernPointsはリスト表示) |
| 人格 | 文体は本人のボイス&トーン(03章)。AIが書いても「わたし」の声で統一 |

**成功指標**: 商品ページの読了率(80%スクロール)30%以上。

## 5.4 アフィリエイトクリック(コンバージョン)

**想定**: レビューを読み納得した状態。摩擦なくYahoo!へ送る。

- CTA配置は3点固定: (1) 商品ヘッダー内(ファーストビュー付近)、(2) 本文読了直後の文末、(3) 記事内 ProductEmbed のボタン。**それ以上増やさない**(信頼>クリック数)。
- ボタン文言は行動が分かる形で統一: 「Yahoo!ショッピングで見る」「Yahoo!トラベルで見る」。「今すぐ購入」等の煽り文言は禁止。
- 外部リンク仕様: `target="_blank"` + `rel="nofollow sponsored noopener"`。リンク切れはビルド時チェック(URL未設定の商品はボタン非表示+ビルド警告)。
- 価格は「購入時価格」と明示し、最新価格はYahoo!側で確認する導線にする(価格追従の保守コストをゼロに)。
- 計測: リンクに内部識別子(pos=product_header 等)を付与。V3でWorkersクリック計測に接続。

**成功指標**: 商品ページ→アフィリエイトCTR 8%以上(業界中央値3〜5%に対し、読了後配置で上振れを狙う)。

## 5.5 SNS導線(双方向)

**サイト→SNS**: フッター+Aboutにのみ設置(記事中にフォローボタンを散らさない)。「Instagramで日常を発信しています」の1行+アイコン。
**SNS→サイト**(主戦場):

| チャネル | 導線 | 運用(AIが文面生成) |
|---|---|---|
| IGストーリー | リンクスタンプ→記事/商品 | 日常投稿の翌日に「詳しくはサイトに書きました」型のストーリーを生成 |
| IGフィード/リール | プロフィールURL+「詳細はプロフィールから」 | キャプション末尾定型文を自動挿入 |
| プロフィールリンク | サイトトップではなく `/articles/`(最新が見える)推奨。キャンペーン時は該当記事へ | リンク先はAIパイプラインの投稿データに含める |
| Threads | 本文にURL直貼り可 | 記事要約+URLの投稿文を生成 |
| TikTok/YouTube | 概要欄URL | 台本生成時に概要欄文面も同時生成 |

- 全SNS流入URLに `?utm_source=instagram&utm_medium=story` 等を付与(生成時に自動)。Cloudflare Web Analyticsで流入比較可能に。

## 5.6 再訪問導線(ブランド化の核心)

**思想**: 再訪は機能では作れない。「また見たい世界観×更新の律動」で作る。機能はそれを補助する。

| 施策 | 内容 | Version |
|---|---|---|
| 更新の律動 | 週1本以上の記事公開(AIパイプラインで担保)。トップの "Journal" が常に動いている | V1〜 |
| ストーリー連動 | 毎日のストーリーが「サイトに詳しく書いた」導線を持つ。SNSフォロワー=再訪問者化 | V2 |
| お気に入り | 端末内お気に入り→「あとで見る」の受け皿 | V2 |
| ニュースレター | 月2回「今月の暮らしと買ってよかったもの」。AIがドラフト生成 | V2 |
| PWA | ホーム画面追加(manifest)。プッシュ通知はやらない(静けさに反する) | V1(manifest)/V3(オフライン) |
| 定点コンテンツ | 「買ってよかった◯選」を毎年更新=毎年見に来る理由 | V2〜 |

**成功指標**: 再訪問率(30日)15%以上、ニュースレター開封率40%以上。

## 5.7 UX非目標(やらないことの明文化)

- 無限スクロール(誌面としての「終わり」を大切にする)
- コメント欄(運用負荷・治安コスト。反応はSNSで受ける)
- 会員登録・パーソナライズ(全員に同じ誌面を見せるのがマガジン)
- ゲーミフィケーション・ポイント(世界観に反する)
