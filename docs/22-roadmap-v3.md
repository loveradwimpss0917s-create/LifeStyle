# 22. V3ロードマップ — 「開店」から「ブランド」へ

> 21章の監査結論に基づく。主語は機能ではなく **①計測 ②生産ライン ③収益導線**。
> 各Phaseに「オーナーがやること(AI代替不可)」を明記する — これを放置するとどの施策も無意味になる。

## Phase 4(〜1年): ブランドにする

**テーマ: 「この人の暮らしが好きだから見に来る」状態を作る。メディアではなくブランドとして認知される。**

| # | 施策 |
|---|---|

| 4-2 | E-E-A-T強化: 著者ページ・実測レビューフォーマット(計測値/使用ログ)・Person構造化データ |

**KPI**: 月次売上の安定黒字 / 指名検索の発生 / 再訪率30%

## 機能追加案の優先順位(S=今すぐ〜C=保留)

| 機能 | 優先 | 判断理由 |
|---|---|---|
| アクセス解析 | **S** | 計測なき最適化は禁止(21章)。しかもprivacy記載との不整合解消が急務 |
| クリック計測(/go/) | **S** | 収益最適化の目。Pages Functionsで静的構成をほぼ保ったまま実現可 |
| お気に入り | **A** | 設計済み(04章§4.8)・実装コスト小・再訪理由になる |
| おすすめ診断 | **A** | 静的決定木で実装可。回遊とCVの両方に効く。「AI診断」の見せ方でブランド演出 |
| 関連記事最適化 | **A** | タグ重み+新着減衰のスコアリング改良。実装コスト小 |
| SNS連携 | **A** | 文面自動生成(Stage4)まで。API自動投稿はV4(審査コスト大) |
| サイト内検索改善 | **A** | Pagefindフィルタ+プレースホルダー。新規エンジン不要 |
| ランキング自動化 | **B** | 既存rating+rankingWeightで動作中。クリック実測導入後に自動重み付けへ |
| 比較機能強化 | **B** | ComparisonAuto実装済み。商品数30超えてから拡張 |
| 管理画面 | **B** | まずGitHub Mobile+PRプレビューで運用(コストゼロ)。専用UIはPhase4(25章) |
| タグ/カテゴリ改善 | **B** | データ駆動済み。商品50超で再編 |
| AI検索 | **C** | Pagefindで十分。ランニングコストと静的構成の破壊に見合わない |
| AIアシスタント(訪問者向け) | **C** | 00章P-7の判断維持。個人メディア規模ではコスト過大 |
| 価格比較(モール横断) | **C** | 00章P-8の判断維持。在庫価格の鮮度管理は静的サイトと相性最悪。V4で再評価 |
