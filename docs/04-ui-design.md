# 04. UI設計(全画面)

全画面共通: ヘッダー(グローバルナビ)+フッター。セクション間は `--section-gap`。ワイヤーフレームはモバイル(M)とデスクトップ(D)の差分を記載。使用コンポーネント名は [06-component-design.md](06-component-design.md) に対応。

## 4.1 トップページ `/`

**目的**: 世界観の提示(=ブランド体験)と、鮮度の証明、各ハブへの導線。**売り場に見せない**。

```
┌────────────────────────────────────┐
│ Header                                        │
├────────────────────────────────────┤
│ ① Hero                                       │
│   フルブリード写真(季節の1枚・16:9)          │
│   英字ラベル "LIFESTACK — Life & Living"       │
│   タグライン「日常を、美しく残す。」           │
│   (テキストは写真下に置く。写真上に重ねない)  │
├────────────────────────────────────┤
│ ② 最新の記事 "Journal"                        │
│   ArticleCard × 3(D: 3列 / M: 1列)           │
│   → すべての記事へ                            │
├────────────────────────────────────┤
│ ③ 編集部のおすすめ "Editor's Picks"           │
│   FeaturedArticle × 1(大判・横組み)           │
│   + ProductCard × 4(D: 4列 / M: 2列)         │
├────────────────────────────────────┤
│ ④ カテゴリ "Explore"                          │
│   CategoryCard × 6(写真+英字名+和名)        │
│   (D: 3列2段 / M: 2列3段)                    │
├────────────────────────────────────┤
│ ⑤ 特集バンド: 旅行 "Travel"                   │
│   背景 --color-bg-secondary                    │
│   旅行カテゴリの記事2本+商品2つ                │
├────────────────────────────────────┤
│ ⑥ 特集バンド: 子育て "Family"(⑤と同構造)    │
├────────────────────────────────────┤
│ ⑦ ランキング "Best Picks"                     │
│   RankingList(1〜5位・番号は Cormorant の数字) │
│   → ランキングページへ                        │
├────────────────────────────────────┤
│ ⑧ About導線                                   │
│   ポートレート or 暮らし写真 + 紹介2文 + リンク  │
├────────────────────────────────────┤
│ Footer                                        │
└────────────────────────────────────┘
```

- Heroの写真は `content/site.json` の `heroImage` で編集可能(季節ごとに差し替え運用)。
- ①〜⑧の並びは「世界観→鮮度→商品→分類→深掘り→人格」の順。商品を最初に出さない。

## 4.2 カテゴリページ `/categories/{slug}/`

**目的**: 生活シーン単位のハブ。記事と商品を1画面で編む(記事一覧と商品一覧を分離しない)。

```
① カテゴリヘッダー: 英字名(Cormorant大)+和名+リード文(2文)+代表写真
② パンくず
③ 特集記事(そのカテゴリのpinned記事1本・大判)
④ 記事グリッド ArticleCard × 最大6 → もっと見る(ページネーション)
⑤ 罫線
⑥ このカテゴリの商品 "Items" ProductCard × 最大8 → /products/?category={slug}
⑦ 関連タグ TagChip 一覧
```

- ソートはUI無し(新着順固定)。フィルタUIもカテゴリページには置かない(静けさ優先)。絞り込みは検索とタグに委譲。

## 4.3 商品ページ `/products/{slug}/`

**目的**: 「友人に見せてもらう」体験。スペック表ではなく実体験。**アフィリエイトの主戦場**。

```
① パンくず(Home > カテゴリ > 商品名)
② 商品ヘッダー
   D: 左55% ギャラリー / 右45% 情報(sticky)
   M: ギャラリー → 情報の縦積み
   ├ Gallery(メイン写真+サムネ3〜5枚。生活シーン写真)
   └ 情報カラム:
      ブランド名(--ink-tertiary・小)
      商品名(h1)
      RatingStars(おすすめ度)+使用期間バッジ「使用6ヶ月」
      リード文(ひとことレビュー・1文)
      価格(参考価格・購入時)
      AffiliateButton「Yahoo!ショッピングで見る」(墨色・full width)
      PR表記(小・ボタン直下)※法令
      TagChip 群
③ 本文 "Review"(--container-text)
   ├ 「なぜ買ったか」(導入)
   ├ 「実際に使ってみて」(体験)
   ├ GoodPoints(良かった点 3〜5・チェックリスト風)
   ├ ConcernPoints(気になった点 1〜3)※必須
   └ 「使用シーン」(写真+キャプション 1〜3)
④ 商品情報表(SpecTable: ブランド/カテゴリ/価格/購入日/使用期間)
⑤ 文末CTA(AffiliateButton 再掲+一言)
⑥ この商品が登場する記事(ArticleCard × 自動逆引き)
⑦ 同じカテゴリの商品(ProductCard × 4)
```

- JSON-LD: `Product` + `Review`(reviewRating=おすすめ度)。
- 「良かった点/気になった点」は frontmatter の構造化データから描画(AI生成→承認済みデータ)。

## 4.4 記事ページ `/articles/{slug}/`

**目的**: マガジンの誌面。読む体験そのもの。5記事タイプ共通レイアウト+タイプ別の差分。

```
① パンくず
② 記事ヘッダー(--container-text)
   カテゴリラベル(英字・アクセント色)
   タイトル(h1)
   リード文 / 公開日・更新日 / タグ
③ ヒーロー画像(--container-wide まで拡幅・3:2)
④ 本文(--container-text・Markdown/MDX)
   使用可能な埋め込み:
   ├ ProductEmbed(商品カード差し込み。record: productId)
   ├ 画像(キャプション付き figure)
   ├ Callout(ひとことメモ)
   └ ComparisonTable(comparison記事)/ RankingList(ranking記事)
⑤ PR表記(商品リンクを含む場合、本文冒頭に1行+②の下に自動)
⑥ 登場した商品まとめ(ProductCard 横スクロール・自動集計)
⑦ 「次に読む」(editorial指定 or 同カテゴリ最新 1本・大判)
⑧ 同カテゴリの記事(ArticleCard × 3)
```

タイプ別差分:
- **ranking**: 本文先頭に目次(1位〜n位アンカー)。各順位は RankingItem(順位数字+ProductEmbed+講評)。
- **comparison**: ComparisonTable(2〜3列・スマホは横スクロール)を上部に。結論を先に書く構成をテンプレート化。
- **log**: PR表記なし(商品リンクがない場合)。写真を多め、ProductEmbedは任意。

## 4.5 プロフィール `/about/`

**目的**: ブランドの人格。E-E-A-Tの根拠。フォロー・ファン化への転換点。

```
① ブランドステートメント(Hero的な1枚+タグライン)
② わたしについて(300字・家族構成/暮らし/なぜ発信するか)
③ このサイトについて(3つの約束: 実際に使ったものだけ/気になった点も書く/広告表記は明確に)
④ SNSリンク(SocialLinks: Instagram×2, YouTube, TikTok, Threads)
⑤ 使用機材・撮影について(写真アカウントとしての矜持・少数)
⑥ お問い合わせ(mailto or フォームURL)+ PRポリシーへのリンク
```

## 4.6 検索 `/search/`

**目的**: 目的型ユーザーの最短経路。Pagefind UIをブランドトークンでスタイル。

```
① 検索入力(大きく1つ・オートフォーカス・placeholder「商品名・カテゴリ・気分で探す」)
② 結果(Pagefind: タイトル/抜粋/サムネ。商品と記事が混在、タイプバッジで区別)
③ 空状態: 人気タグ TagChip 群+カテゴリ CategoryCard(行き止まり防止)
```

- ヘッダーの検索アイコン → `/search/` へ遷移(モーダルにしない。シンプル&ディープリンク可能)。
- Pagefindはビルド時インデックス。日本語は`lang="ja"`で分かち書き対応。

## 4.7 ランキング `/ranking/`

**目的**: 「結局どれがいいの?」への即答。シェアされる定番ページ。

```
① ページヘッダー "Best Picks" + 更新日 + 選定基準1文(信頼性)
② 総合 Top 10(RankingItem × 10)
   各: 順位(Cormorant数字・大)/ 写真 / 商品名 / ひとこと / おすすめ度 / →商品ページ
③ カテゴリ別 Best 3(カテゴリごとに RankingList 小)
④ PR表記
```

- 順位は商品frontmatterの `rating` + `rankingWeight`(編集調整値)で決定。ビルド時計算。

## 4.8 お気に入り `/favorites/`(V2)

**目的**: 再訪の理由を作る。端末内完結・ログイン無し。

```
① ページヘッダー「お気に入り」+説明1文(この端末にのみ保存されます)
② お気に入り商品グリッド(ProductCard × n)
③ 空状態: ハートアイコンの説明+ランキングへの導線
```

- 実装: `localStorage` キー `lifestack:favorites` に productId 配列。FavoriteButton(ハート)は商品カード右上・商品ページに設置。island(client:load最小JS)。
- カード表示は productId → ビルド時出力の `/api/products-index.json`(静的JSON)を fetch して描画。

## 4.9 共通画面

- **404**: 「ページが見つかりません」+検索ボックス+カテゴリ一覧。トーンは静かに。
- **タグページ `/tags/{slug}/`**: タグ名ヘッダー+該当の記事/商品混在グリッド。
- **`/disclosure/` `/privacy/`**: テキストページテンプレート(--container-text)。
