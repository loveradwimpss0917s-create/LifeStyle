# 15. アイコンシステム — HIBISTACK

> 現行 `src/components/base/Icon.astro`(単一コンポーネント・インラインSVG方式)を拡張する。
> 方式は変更しない(スプライト外部ファイル化はHTTPリクエストとキャッシュ管理が増えるだけで、
> 現規模ではインラインが最軽量)。

## 1. 造形ルール(全アイコン共通)

| 項目 | 値 |
|---|---|
| viewBox | 24×24 |
| セーフエリア | 外周2px(描画は2〜22の範囲) |
| 線幅 | 1.5(fill系は例外。星・ハート塗りつぶし時のみfill) |
| 端点/角 | round / round |
| 色 | `currentColor` のみ。多色禁止 |
| スタイル | 幾何学的・最小ストローク数。装飾・影・グラデ禁止 |
| 角丸感 | ロゴシンボル(14章)と同じ「角丸矩形+線」の語彙を優先する |

**実装上の必須事項**: 親コンポーネントからIconへ渡したclassにはAstroのスコープ属性が付かない。
色指定は必ず `:global()` で書くこと(RatingStars/GoodPoints/ConcernPointsで実際に起きた既知バグ。該当コミット参照)。

## 3. 使用ルール

- サイズは 16 / 18 / 20 / 24 のみ(`size` prop)。中間値禁止
- テキスト隣接時は `gap: var(--space-2)`、垂直中央揃え
- 単独でインタラクティブにする場合、タップ領域は最小44×44px(ボタン側でpadding確保)し、`aria-label` を必ず付与
- 装飾目的のアイコンは `aria-hidden="true"`(Icon.astroが既に付与済み)
- SNS実ブランドアイコンは公式ガイドラインの改変禁止に配慮し、単色・簡略アウトライン表現に留める(ボタン背景に各社ブランドカラーを敷かない)

## 4. 実装形式

- `Icon.astro` の `IconName` union型に新規名を追加し、条件分岐でpathを追加する(現行方式踏襲)
- arrow系は1つのpath定義を `transform="rotate(...)"` で使い回し、コード重複を避ける
- 追加後、全アイコンを並べた確認用ページは作らない(ビルド成果物を汚さない)。確認はStorybook等も使わず、一時的なローカルページで行い削除する
