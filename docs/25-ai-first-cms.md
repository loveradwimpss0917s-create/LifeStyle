# 25. AIファーストCMS設計

> 「人が編集するCMS」ではなく「**AIが書き、人が承認する受信箱**」。
> WordPress的な編集画面は作らない — 編集画面が必要になる時点でAIファーストの敗北とみなす。

## 1. 設計原則

1. **Gitが唯一のデータベース。** CMSはGitの上のUIに過ぎない。DB二重化はしない(静的サイト+PRレビューの利点を全部失うため)
2. **人間のプライマリUIは「編集フォーム」ではなく「受信箱(Inbox)」。** 起案・執筆・整形はすべてAIの仕事。人間の操作は 承認 / 修正指示 / 事実の提供 の3種だけ
3. **入力は自然言語+写真のみ。** frontmatterやMarkdownをオーナーに書かせたら設計ミス
4. **段階導入。** 専用アプリは最後。まず既存インフラ(GitHub Mobile)で同じ体験を作り、価値検証してから作り込む

## 2. アーキテクチャ(3層)

```
┌─ Intake層(入力) ─────────────────────────────┐
│ iPhoneショートカット「HIBISTACKに送る」                    │
│  写真1〜6枚+ひとことメモ → GitHub Issue(label: intake)   │
│  ※将来: LINE Bot / メール転送も同じIssueフォーマットへ正規化 │
└──────────────┬───────────────────────┘
               ▼
┌─ Agent層(生成・GitHub Actions + Claude API) ──────────┐
│ analyze → compose → seo → open-pr(09章Stage1-6)           │
│ 定期job: rewrite候補 / リンク死活 / 週次レポート / SNS再掲候補   │
│ プロンプトはpipeline/prompts/でバージョン管理                  │
└──────────────┬───────────────────────┘
               ▼
┌─ Review層(承認) ─────────────────────────────┐
│ V3: GitHub Mobile + Cloudflareプレビュー URL                  │
│     (PR = 受信箱。承認=Merge。修正指示=コメント→Actionsが再生成) │
│ V4: 専用受信箱アプリ(下記§3)                                │
└──────────────────────────────────────┘
```

## 3. V4専用アプリ「HIBISTACK Desk」(受信箱アプリ)

GitHub Mobileの体験で不足が実証された場合のみ着手する。

| 項目 | 設計 |
|---|---|
| ホスティング | Cloudflare Pages(本体とは別プロジェクト)+ Pages Functions |
| 認証 | GitHub OAuth(オーナー1名のみ許可。Fine-grained権限) |
| データ | GitHub API経由でPR/Issue/contentsを直接読む(独自DBなし) |
| 画面1: Inbox | 承認待ちPR一覧。カード=プレビュー画像+タイトル+チェックリスト残数。スワイプで承認(merge)/ タップで詳細 |
| 画面2: 修正指示 | 自然言語でコメント→ Actionsが該当ステージのみ再生成してPR更新 |
| 画面3: 送信 | Intakeフォーム(写真+メモ)。ショートカットの代替 |
| 画面4: ダッシュボード | 週次レポート(PV/クリック/売上)+ リンク健全性 + 公開カレンダー |
| 通知 | PR作成時にプッシュ(GitHub通知 or Web Push) |

**作らないもの**: Markdownエディタ、メディアライブラリ管理画面、ユーザー管理、コメント管理。
これらが欲しくなったら要求をAgent層の自動化に翻訳できないか先に検討する。

## 4. 段階導入ロードマップ

| 段階 | 内容 | 判断基準 |
|---|---|---|
| V3(Phase2) | パイプライン+GitHub Mobile運用 | まずこれで週1時間運用を実証 |
| V3.5(Phase3) | ダッシュボードのみ静的ページ化(自動レポートをサイト外の非公開ページに) | レポートIssueが読みにくくなったら |
| V4(Phase4) | HIBISTACK Desk着手 | GitHub Mobileでの承認が月50件を超え、UI摩擦が律速になったら |
