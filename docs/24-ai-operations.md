# 24. AI運営戦略 — 自動化レベル定義

> 09章(パイプライン設計)を運営全体へ拡張する。原則は不変:
> **AIは編集部、オーナーは編集長。公開は常に人間の承認(PRマージ)を経る。事実の捏造は絶対禁止。**

## 1. 自動化レベルの定義

| レベル | 意味 |
|---|---|
| L0 | 人間のみ(AI関与なし) |
| L1 | AIが補助(人間が主、AIが提案) |
| L2 | AIが主、人間が承認(生成→レビュー→マージ) |
| L3 | 全自動(人間は事後確認のみ) |

## 2. 業務別の自動化方針

| 業務 | 目標 | 方針 |
|---|---|---|
| 商品データ起こし(名称/カテゴリ/タグ/alt) | **L2** | 写真+メモから解析。確信度低い項目はTODOマーカーでPRに残す(09章Stage1) |
| レビュー本文執筆 | **L2** | **体験の事実はメモ由来のみ**。メモにない体験談は書かず`CONFIRM`マーカーで人間に確認(09章§7)。文体はvoice.md+few-shotで統一 |
| 記事(まとめ/比較/ランキング)執筆 | **L2** | 既存商品データの再編集なので捏造リスクが低く、自動化効率が最も高い領域 |
| SEOメタ(title/description) | **L3** | 生成+CIでの文字数・重複検証まで自動。人間レビューはPR全体で兼ねる |
| 内部リンク提案 | **L2** | 既存記事から関連3件を提案しPRに含める。無関係リンクの機械的な埋め込みは禁止 |
| SNS投稿文面(6チャネル) | **L2** | 生成は自動、投稿は手動コピー(当面)。API自動投稿はV4判断 |
| 画像 | **分割** | OGP=既にL3(satori)。**商品・体験写真=L0で恒久固定**(実写真がブランドの核・E-E-A-T)。装飾イラスト・カテゴリイメージのみAI可(19章の`ai-`規約) |
| リライト | **L2** | 90日経過×流入低下の記事を週次で候補Issue化→承認後にAIが改稿PR |
| 分析レポート | **L3** | 週次でPV/クリック/売上をIssueに自動投稿。人間は読むだけ |
| アフィリエイトURL貼付 | **L0で恒久固定** | ASP規約上、リンク発行は本人の管理画面から。PRのTODOチェックボックスで促す(09章Stage6) |
| リンク死活・期限監視 | **L3** | CI+週次チェック。切れリンクはIssue自動起票 |
| コメント・問い合わせ対応 | L0 | 現状フォーム無し。導入時も人間対応 |

## 3. オーナーの作業(自動化後の姿)

```
日次: ストーリー投稿のついでに「写真+ひとことメモ」を送る(1分)
随時: 届いたPRをスマホでプレビュー確認 → 修正指示 or マージ(3分)
      アフィリエイトURLを貼る(2分)
週次: 自動レポートを読む(5分)/ SNS文面をコピーして投稿(10分)
```

**合計: 週1時間以内。** これを超える運用は設計ミスとみなして自動化を追加する。

## 4. ガードレール(09章§7の再掲+追加)

1. 事実の捏造禁止: 体験談はメモ由来のみ。スキーマ(purchasedAt必須・concernPoints必須)が構造的に担保
2. 公開は常にPRマージ=人間の操作。全自動公開は存在しない
3. プロンプトインジェクション対策: intakeメモはデリミタ分離してデータとして扱う
4. 生成物には必ず自己検証チェックリスト(Zod相当)を通す
5. **AI画像を実体験の証拠として使わない**(19章)。altに「イメージ写真」明記
6. 薬機法・景表法NGワードのCIチェック(禁止語リストをvoice.mdに機械可読で持つ)
