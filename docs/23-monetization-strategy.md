# 23. 収益化戦略 — V3

> 原則(00章§6)は不変: **収益はブランドの副産物。CTAは商品ページ主1+文末1まで。バナー・ポップアップは永久禁止。**
> V3はこの原則の内側で「本数×クリック率×成約率×単価」の各係数を上げる。

## 1. 収益方程式と現在地

```
月間収益 = 流入 × アフィリエイトクリック率 × 成約率 × 平均報酬
現在:     ほぼ0 ×  計測不能            × 不明   × Yahoo!のみ
```

**最初の仕事は式の各項を「測れる」ようにすること**(/go/クリック計測+CF Web Analytics)。最適化はその後。

## 2. チャネル別戦略

### 2.1 Yahoo!ショッピング(主軸・維持)
- 競合が楽天/Amazonより薄い戦略優位(00章SWOT)は不変。主軸を動かさない
- 5のつく日・買う買うサンデー等の時限文脈は**SNS側で消化**し、サイトは非時限の資産記事に徹する(既定方針の再確認)
- リンク健全性: `checkedAt`180日警告(実装済み)+HTTP死活チェックをCIに追加(V3)

### 2.2 Yahoo!トラベル(第2の柱に育成)
- 商品型ではなく**記事型**が主戦場: 「子連れ×宿レビュー」「モデルコース」フォーマットを確立
- 旅行記(log)→宿詳細(review)→Yahoo!トラベルCTAの2段導線。旅行記自体はCTA無しでE-E-A-Tと回遊に貢献
- 季節性が強いため、特集ハブ(夏旅/年末年始)と組み合わせて再掲運用

### 2.3 Amazon・楽天(複線化・Phase 3)
- **もしもアフィリエイト経由**で一括導入(個別審査より速く、W報酬制度あり)。スキーマは対応済み(amazon/rakuten フィールド既設)
- 表示は「主ボタン=Yahoo!、下に小さくAmazon/楽天テキストリンク」。主従を崩さない(ボタンを3つ並べない)
- 読者の主要モールが違うだけで取りこぼす分を回収するのが目的。報酬率での主客転倒はさせない

### 2.4 ディスプレイ広告(AdSense等)— **非採用を明言**
- 単価の低いバナーでブランド(=最大の資産)を毀損するのは割に合わない。Kinfolk的世界観と両立しない
- 再検討条件: 月間10万PV超かつアフィリエイト成長が頭打ちになった場合のみ(おそらく来ない)

### 2.5 提供品・タイアップ(Phase 4)
- 受け入れ基準を先に公開してから開始(13章§8): 実際に使う・正直に書く・明示する
- 単価はアフィリエイトの10〜100倍。ブランドが立ってからの本命収益源

### 2.6 SNS
- 直接収益化はしない。役割は①サイトへの導線 ②時限セール文脈の消化 ③指名検索の醸成
- 文面はAI生成(24章)、投稿は当面手動。API自動投稿はV4判断

## 3. サイト内の導線最適化(データ取得後)

1. `data-pos`別クリック率の計測(product_header / product_footer / product_sticky / ranking …)
2. 低CTR位置の文言・配置をABではなく**逐次改善**(トラフィック規模的にABテストは統計不能。before/after比較で十分)
3. ランキングの`rankingWeight`をクリック実測で補正(手動→将来自動提案)
4. 診断(/quiz/)からの遷移率を計測し、質問設計を改善

## 4. やらないことリスト(明文化)

| 施策 | 理由 |
|---|---|
| バナー広告・ポップアップ | 永久禁止(00章)。信頼=LTVを毀損 |
| 価格比較の自動更新 | 静的サイトと相性最悪。誤価格表示は信頼を一撃で壊す |
| 「最安値」「今だけ」等の煽り文言 | 13章ライティングガイド違反 |
| リンクだけの商品紹介(未購入品) | ブランドの核への違反。スキーマ(purchasedAt必須)でも防止済み |
