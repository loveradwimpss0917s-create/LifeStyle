# 13. ブランドアイデンティティ — HIBISTACK

> 本書はブランドの「正」である。V1で仮称としていたサイト名は **HIBISTACK** に正式決定した。
> 以後、コピー・デザイン・実装のすべてはこの章のトーンに従う。00章のブランド戦略を継承し、これを上書きする。

## 1. ネーミング

| 要素 | 意味 |
|---|---|
| **HIBI(日々)** | 特別な日ではなく、繰り返される毎日。ストーリー投稿という日課。生活そのもの |
| **STACK(積む)** | 日々の記録が資産として積み重なっていくこと。コンテンツの複利。暮らしの道具が少しずつ揃っていくこと |
| **HIBISTACK** | 「日々を、積む。」— 毎日の小さな記録・小さな買い物・小さな工夫の積み重ねが、豊かな暮らしと一つのブランドになる |

発音: ヒビスタック。表記は常に大文字 `HIBISTACK`(和文中でも)。`HibiStack` `hibistack` は不可(コード内識別子を除く)。

## 2. ブランドメッセージ(タグライン)

> **日々の暮らしを、少しだけ豊かにする。**

- 英語版: **Stacking better days.**
- 「少しだけ」が核。誇張しない・煽らない・買わせようとしない。1つの道具で生活は劇的には変わらないが、確かに少し良くなる——その正直さがブランドの信頼になる。
- 旧タグライン「日常を、美しく残す。」は退役(実装上は `src/content/site.json` の `tagline` / `taglineEn` を差し替える)。

## 3. ブランドストーリー

毎日ストーリーを投稿するふたつのInstagramアカウントから、このブランドは始まった。
旅の記録、子どもとの時間、朝のコーヒー、買ってよかったもの。流れて消えていくはずだった日々の断片を、
一つずつ積み上げて残していく場所。それがHIBISTACK。

紹介するのは、実際に買って、使って、暮らしの中で確かめたものだけ。
良かった点だけでなく、気になった点も書く。
劇的なビフォーアフターではなく、「ちょっと家事が楽になった」「朝が少し楽しみになった」という等身大の変化を記録する。

## 4. 理念・ミッション・ビジョン

| 項目 | 内容 |
|---|---|
| 理念(Purpose) | 暮らしは、記録すると豊かになる |
| ミッション | 実体験に基づく正直な記録を積み重ね、読者が「自分の暮らしに合うもの」を安心して選べるようにする |
| ビジョン | 「この人の暮らしが好きだから見に来る」個人発のライフスタイルブランドとして、検索にもSNSにも依存しない読者との関係を築く |
| 提供価値 | ①実体験の一次情報 ②世界観(見ていて心地よい) ③選択の時短(正直なレビューで迷いを減らす) |

## 5. ターゲット

### プライマリペルソナ「ハルカ」(32)
- 未就学児のいる共働き世帯。都市近郊在住
- Instagramで暮らし系アカウントをフォロー。買い物前に必ずレビューを検索する
- 広告っぽいアフィサイトを見分けて離脱する。「本当に使った人の感想」を求めている
- 求めるもの: 時短家電・育児用品・旅行先の実体験情報

### セカンダリペルソナ「ケンジ」(38)
- ガジェット・コーヒー器具好き。デザインの良い道具に囲まれたい
- 比較記事を読み込み、納得してから買うタイプ
- 求めるもの: スペックと使用感の両方。誇張のない評価

### 共通する態度
「安いから買う」ではなく「良いものを長く使いたい」。価格より納得感。世界観に共感すればリピーターになる。

## 7. ライティングガイド

### 7.1 文体
- です・ます調。一人称は「わたし」(ひらがな)。読者への呼びかけは「〜な方」
- 一文は60字以内を目安。段落は3〜4文まで。結論を先に書く
- 体験は必ず具体で書く: ×「とても便利」→ ○「椅子をどかさずに掃除機がかけられるようになった」

### 7.2 禁止・回避表現
| 区分 | 例 |
|---|---|
| 誇張(禁止) | 神・最強・究極・革命・爆売れ・人生が変わる・No.1(根拠なし) |
| 断定の効能(禁止・薬機法/景表法) | 痩せる・治る・肌が若返る・絶対に・必ず |
| 煽り(禁止) | 今すぐ・急いで・売り切れ注意・買わないと損 |
| 空虚な形容(回避) | とても・すごく・かなり(→具体的な体験に置換) |
| 絵文字 | 本文では使用しない。SNS派生文面のみ可(1投稿2個まで) |

### 7.3 数値・事実の扱い
- 価格は「購入時価格」と明記し、最新価格はリンク先で確認するよう促す
- 使用期間・購入日は必ず書く(frontmatterの `usagePeriod` / `purchasedAt` が担保)
- 体験していないことは書かない。推測は「〜かもしれません」と明示する
- 気になった点(concernPoints)を最低1つ必ず書く(スキーマで強制済み)

### 7.4 定型文
- PR表記: 「※本サイトはアフィリエイト広告を利用しています。」(コンポーネント固定文言・変更禁止)
- CTA文言: 「Yahoo!ショッピングで見る」(「購入はこちら!」等の煽り形は不可)
- 記事末の締め: 次の行動を1つだけ提示(関連記事1本 or 商品ページ)。複数のCTAを並べない

## 8. ブランド運用ルール

1. 収益はブランドの副産物(00章§6踏襲)。CTAは商品ページにつき主1+文末1まで。バナー・ポップアップは永久禁止
2. 「人気」表記はV1では編集部選定(editorsPicks / rankingWeight)を意味する。クリック実測に基づく「人気」はV3の計測導入後にのみ使用可
3. 提供品・タイアップは現時点で扱わない。扱う場合は記事冒頭に明示(10章§2.2)
4. 写真は実写真を正とする。プレースホルダーSVGは実写真が用意でき次第置き換える
